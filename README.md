# 시간표 랜덤 추천기

개설된 강의에 한하여 시간표를 가져와, 사용자가 제시한 조건에 맞는 시간표를 랜덤으로 생성합니다.
그렇게 된다면 추후 시간표를 맞춰볼 필요없이 자동으로 만들어진 것에서 골라서 수강신청을하면 된다.
특히 해당과목의 수강신청이 실패했을 경우에 추천하는 대체과목도 알려주도록 설정한다.





## 설계

Java Swing 기반 데스크탑 애플리케이션을 GUL(레이아웃 중심 구조) 방식으로 설계 및 구현해야함.

프로젝트 이름: 시간표 추천 시스템 (Timetable Recommendation System)

---

## 🎯 목표
사용자가 선택한 개설 과목들만을 기반으로 가능한 모든 시간표 조합을 생성하고,
시간 충돌 검사 및 제약조건을 적용하여 최적의 시간표 5개를 추천하는 시스템을 구현한다.

랜덤 생성이 아니라 "사용자 선택 기반 + 제약조건 + 점수 기반 추천 시스템"이다.

---

## 🧱 전체 UI 구조 (GUL Swing 기반)

### 1. 전체 레이아웃
- BorderLayout 사용

---

### 2. NORTH (상단 패널)
- FlowLayout 또는 GridLayout(1,n)
- 구성:
  - 전략 선택 JComboBox
    (금공강 우선 / 월공강 우선 / 최대학점 / 1교시 배척 / 전필 우선 / 교필 우선)
  - [시간표 생성] JButton

---

### 3. CENTER (핵심 영역)
JSplitPane (Horizontal Split)

#### LEFT PANEL (개설과목 + 필터)
GridBagLayout 사용

구성:
- 학과 검색 JTextField (자동완성 지원)
  - 예: "컴공" → "컴퓨터공학과"
  - 예: "행정" → "행정학과"
- 과목 구분 JComboBox
  - 전공필수 / 전공선택 / 교양필수 / 교양선택
- 학점 필터 (1~6)
- 개설과목 리스트 (JCheckBox 또는 JList)
- 선택 버튼 또는 체크 기반 선택

---

#### RIGHT PANEL (추천 결과)
JList 또는 JTable

- 추천 시간표 5개 출력
- 각 항목:
  - 시간표 이름 (A~E)
  - 점수 표시 (예: 92점)
  - 학점 표시 (예: 17~18학점)

---

### 4. SOUTH (하단 상세 패널)
GridLayout(1,2)

#### LEFT SIDE
- 선택된 시간표 시간표 JTable
  (요일별 시간표 시각화)

#### RIGHT SIDE
- 과목 상세 정보 영역 (JTextArea)
  - 과목명
  - 교수
  - 학점
  - 전필/교필 여부
  - 강의실
- 대체 과목 추천 JList
  - 유사 과목 리스트 출력

---

## 📚 핵심 기능

### 1. 개설과목 조회 및 필터링
- 학과, 과목구분, 학점 기준 필터링
- 자동완성 기능 포함 (prefix matching 기반)

---

### 2. 사용자 선택 기반 과목 설정
- 사용자가 체크한 과목만 사용
- 선택된 과목 리스트 기반으로 시간표 생성

---

### 3. 시간표 생성 (핵심 알고리즘)
- 백트래킹 또는 DFS 기반 조합 생성
- 선택된 과목으로만 시간표 구성
- 시간 충돌 발생 시 제외

---

### 4. 제약 조건
- 목표 학점 기준:
  - 허용 범위: (target - 1) ~ target
  - 초과 학점은 절대 불가
- 1교시 포함 여부 전략 적용
- 전략별 조건 반영

---

### 5. 전략 기반 추천 시스템
사용자가 선택한 전략에 따라 점수 계산:

- 금공강 우선
- 월공강 우선
- 최대학점 수강
- 1교시 최소화
- 1교시 배척
- 전공필수 우선
- 교양필수 우선

---

### 6. 점수 시스템
각 시간표에 대해 점수 계산:

- 공강 많음 (+)
- 연강 적음 (+)
- 1교시 없음 (+)
- 전략 조건 만족 (+)
- 시간 충돌 없음 (+)

상위 5개 시간표 추천

---

### 7. 상세 기능

#### 시간표 클릭 시
- JTable에 시간표 출력
- 해당 과목 리스트 표시

#### 과목 클릭 시
- 과목 상세 정보 출력

#### 수강 실패 시
- 대체 과목 추천 기능
  - 같은 학과
  - 같은 과목 구분
  - 유사 과목 기반 추천

## ⚙️ 구현 요구사항
- Java Swing 기반 GUI
- GUL(Layout Manager 중심) 구조 필수
- BorderLayout + GridBagLayout + GridLayout 활용
- 객체지향 설계 필수
- 백트래킹 기반 시간표 생성
- 시간 충돌 검사 로직 포함
- 전략 기반 점수 시스템 구현

---

## 🚀 핵심 요구사항 (중요)
- 선택된 과목만으로 시간표 생성
- 랜덤이 아니라 제약조건 기반 조합 생성
- 모든 시간표는 시간 충돌이 없어야 함
- 학점은 (target - 1 ~ target) 범위만 허용
- 추천 이유를 점수 breakdown으로 표시할 것
